# SNSTK Studio Style Guide

Version: 1.0
Last Updated: July 2026

---

# Purpose

This document defines the visual identity, project organization, and coding conventions for SNSTK Studio.

The goal is consistency. Every screen, icon, asset, and feature should feel like part of the same application.

---

# Brand

Application Name

SNSTK Studio

Tagline

Create native Supernote sticker collections.

Mascot

Nib

Nib is the official SNSTK Studio mascot.

Nib represents:

• creativity
• organization
• simplicity
• productivity
• friendliness

Nib should always appear calm, helpful, and encouraging.

Avoid exaggerated or distracting expressions.

# User Experience

Every feature should answer one question:

"Does this make creating Supernote stickers easier?"

If not, reconsider adding it.

SNSTK Studio should never become cluttered.

Simplicity wins.

# Guiding Principle

SNSTK Studio should feel like a polished desktop application that quietly helps people create beautiful Supernote sticker collections.

The software should never compete for attention.

It should simply help users create.

---
# Development Philosophy

Build one feature at a time.

Finish each feature before beginning the next.

Prefer simple, maintainable solutions over clever ones.

When new ideas arise, add them to the backlog instead of interrupting the current milestone.

Every release should leave the application in a working state.
